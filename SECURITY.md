# Política de Segurança

## Escopo
Este repositório é público e orientado a planejamento técnico.

## Não publicar
- credenciais reais;
- tokens de API;
- chaves privadas;
- endpoints internos não públicos;
- dados de clientes/produção.

## Boas práticas obrigatórias
- usar placeholders e `.env.example`;
- revisar diffs antes de commit;
- usar princípio de mínimo privilégio em tokens;
- rotacionar segredos em caso de vazamento.

## Reporte de vulnerabilidades
Para reportar vulnerabilidades, use canal privado com os mantenedores e inclua:
- descrição do problema;
- impacto;
- passos para reproduzir;
- sugestão de mitigação.

## SLA de resposta (alvo)
- triagem inicial: até 72h;
- plano de ação: até 7 dias.
