# Contribuindo

Obrigado por contribuir.

## Como contribuir
1. Abra uma issue descrevendo contexto e proposta.
2. Alinhe o escopo antes de mudanças grandes.
3. Faça um fork e abra PR com descrição clara.
4. Relacione PR com issue.

## Padrões
- mantenha mudanças focadas;
- documente decisões técnicas;
- evite incluir dados sensíveis;
- preserve compatibilidade com o escopo de planejamento.

## Regras de segurança
- nunca commitar tokens, credenciais ou chaves;
- use placeholders em qualquer configuração;
- reporte falhas de segurança via [SECURITY.md](./SECURITY.md).

## Mensagens de commit (sugestão)
- `docs: ...`
- `chore: ...`
- `plan: ...`
- `adr: ...`

## Checklist de PR
- [ ] Sem credenciais sensíveis
- [ ] Documentação atualizada
- [ ] Escopo alinhado com roadmap
- [ ] Riscos/limites descritos (quando aplicável)
